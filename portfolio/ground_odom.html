Stereo Odometry on Aerial Vehicles


Guide: Dr. Sebastian Scherer, Robotics Institute, Carnegie Mellon University


For aerial vehicles, odometry is often done by using sensors like the Pixhawk PX4FLOW (rightmost in the picture below), which use a single camera doing correlation-based tracking with a sonar for odometry. This has many disadvantages, like small camera field of view (small maximum speeds), bad sonar readings at low range (during take-off), requirement of an inertial unit for angle measurement and height dependent camera focus.


We aimed to replace this with a small-baseline stereo camera (center in picture above). The problem, however, is that tradtional methods of stereo odometry like corner matching do not work because of the relatively texture-less nature of the ground, so one is restricted to correlation-based methods.

With the field of view parallel to the baseline, the height of the vehicle is obtained from a robust estimate of horizontal disparity. A patch-based correlation-maximizing method is used to get a set of disparity measurements and a robust parameter selection algorithm like RANSAC is used to obtain the approximate height.

Alternatively, height, pitch and roll are jointly estimated using a robust correlation-maximizing gradient-descent homography fit between stereo pairs. Using this data, rigid tracking across frames is used to measure position. We obtained better height estimates, maximum speeds and comparable accuracy without an inertial unit as compared to the PX4FLOW.
